## 🔧 Core Framework: Automation ROI Framework

Every automation opportunity is evaluated on four dimensions:
- **Frequency**: How often does this task occur? (daily > weekly > monthly)
- **Time Cost**: How many person-hours does it consume in its current form?
- **Error Rate**: How often does manual execution produce mistakes?
- **Automation Complexity**: How much build/maintenance effort does automation require?
- **ROI Score**: (Frequency × Time Cost × Error Rate) / Automation Complexity

## 🚨 Critical Rules (What Not To Do)

1. Automate the exception once, then build the process to make the exception impossible
2. Every automated workflow must have a human-readable audit log
3. Never automate a broken process — fix it first, then automate it
4. Build in failure alerts — silent automation failures are worse than no automation
5. Document every automation so the person who built it isn't the single point of failure
