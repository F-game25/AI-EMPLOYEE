## 🔄 Growth Experiment Framework

### ICE Scoring
Rate each experiment on:
- **Impact** (1-10): Expected effect on North Star metric if it works
- **Confidence** (1-10): Evidence that this will work (data, analogues, theory)
- **Ease** (1-10): Speed and resource efficiency of implementation
- **ICE Score** = (Impact + Confidence + Ease) / 3

### Experiment Template
```markdown
## Experiment: [Name]
**Hypothesis**: If we [change X], then [metric Y] will [increase/decrease] by [Z%] because [reason]
**ICE Score**: Impact [X] + Confidence [X] + Ease [X] = [Score]
**Primary Metric**: [What we're measuring]
**Guard Rail Metrics**: [What we can't hurt]
**Sample Size Needed**: [Based on MDE and current baseline]
**Test Duration**: [Days]
**Status**: Backlog / Running / Complete
**Result**: [Winner/Loser/Inconclusive] — [% change, p-value]
**Learning**: [What we learned regardless of outcome]
```

## 🌊 Growth Loop Design

### Types of Growth Loops
1. **Viral Loop**: User invites others as core product mechanic
2. **Content Loop**: Users create content → SEO/social traffic → new users
3. **Paid Loop**: Revenue → ad spend → new users → more revenue
4. **Community Loop**: Users engage → community value → new users join
5. **Product Loop**: More users → better product → more users (network effects)

### Viral Coefficient Formula
K = (% users who invite) × (% invitees who convert)
- K > 1.0: Viral growth (product grows on its own)
- K = 0.5-1.0: Assisted viral (paid or other channels needed)
- K < 0.5: Non-viral (pure paid/SEO dependent)
