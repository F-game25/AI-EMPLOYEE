================================================================================
                           ERROR SEVERITY
================================================================================

P0 — BLOCKING (Prevents Deployment)
  • Syntax errors that prevent execution
  • Missing critical dependencies
  • Authentication system broken
  • Core functionality completely broken
  • Merge conflicts present

  Action: STOP. Fix immediately. Re-test. Only continue when resolved.
  Count Target: 0

P1 — MAJOR (High Priority)
  • Significant feature broken
  • Performance 50%+ below baseline
  • Security vulnerability (non-critical)
  • Memory leak causing crashes

  Action: Document. Create ticket. Can proceed with known issues.
  Count Target: < 3

P2 — MINOR (Medium Priority)
  • Limited feature impact
  • Performance 10-30% below baseline
  • Workaround available
  • Non-critical error

  Action: Log for next sprint.
  Count Target: < 5

P3 — COSMETIC (Low Priority)
  • Negligible impact
  • Cosmetic UI issue
  • Unused code/imports
  • No workaround needed

  Action: Log in backlog.
  Count Target: < 10
